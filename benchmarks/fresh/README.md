# Fresh Benchmarks

Use this suite for recent real repo tasks that were collected, curated, and normalized into deterministic fixture-backed benchmark cases.

- `candidates/`: newly collected real-task intake artifacts awaiting or recording review.
- `cases/`: accepted benchmark cases with explicit provenance, graders, and success criteria.
- `rejected/`: candidates that stay recorded for auditability but are intentionally excluded from the runnable corpus.

Do not promote a fresh task into `cases/` unless the source is recent, the scope is clear, the fixture/spec is reproducible, and the expected outcome is human-curated rather than copied from the current system output.
